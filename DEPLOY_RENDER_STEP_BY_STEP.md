# نشر BaytAI على Render — خطوة بخطوة

## 1) ارفع المشروع إلى GitHub
- فك ضغط المشروع.
- أنشئ مستودعًا جديدًا في GitHub باسم `baytai`.
- ارفع جميع الملفات، وتأكد من وجود `render.yaml` في الجذر.

## 2) أنشئ حساب Render
- ادخل إلى Render وسجل الدخول.
- اربط حساب GitHub الخاص بك.

## 3) أنشئ خدمة جديدة من نوع Blueprint
- من لوحة Render اختر: **New > Blueprint**.
- اختر مستودع `baytai`.
- سيقرأ Render ملف `render.yaml` تلقائيًا.
- اضغط **Apply** أو **Create New Resources**.

## 4) انتظر اكتمال البناء والنشر
- سيشغل Render:
  - Build Command: `pip install -r requirements.txt`
  - Start Command: `uvicorn app:app --host 0.0.0.0 --port $PORT`
- بعد اكتمال النشر ستحصل على رابط عام بالشكل:
  - `https://baytai.onrender.com`
  - قد يضيف Render لاحقة إذا كان الاسم مستخدمًا.

## 5) اختبر المنصة
- افتح الرابط العام.
- جرّب إدخال قصيدة ثم اضغط **توليد المحتوى**.
- لاختبار الصحة التقنية افتح:
  - `/healthz`

## 6) لو أردت لاحقًا ربط الذكاء الاصطناعي الحقيقي
أضف مفاتيح API من Render:
- افتح الخدمة > **Environment**.
- أضف متغيرات مثل:
  - `OPENAI_API_KEY`
  - `ELEVENLABS_API_KEY`
  - `RUNWAY_API_KEY`

## 7) عند كل تعديل
- عدّل الملفات في GitHub.
- نفّذ Commit ثم Push.
- Render سيعيد النشر تلقائيًا.

## 8) ملاحظات مهمة
- الخطة المجانية مناسبة للتحكيم والعرض الأولي.
- قد يتوقف التطبيق بعد فترة خمول ثم يستغرق بضع ثوانٍ ليستيقظ.
- إذا أردت رابطًا أقوى وأسرع، يمكن لاحقًا الترقية إلى خطة مدفوعة.
